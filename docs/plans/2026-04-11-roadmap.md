# Hermes Tailchat Roadmap Notes

Context gathered from the user:
- single-user only; no need to scale beyond one user
- exploratory/prototype; optimize for iteration speed
- top wishlist items:
  - mobile-friendly UX
  - intermittent access resilience when another app is in focus or screen is off
  - background work that continues server-side
  - keeping the project up-to-date and following best practices such as Dependabot and SAST

Implications for prioritization:
- prioritize usability and reliability over multi-user architecture
- keep auth/access-control lightweight and tailnet-compatible
- prefer incremental hardening rather than heavy productization
- invest early in regression protection for the specific things that matter now: mobile access, reconnect behavior, background jobs, and deployment hygiene

Suggested roadmap themes:
1. Operationalize the current working `/hermes` deployment path safely and document how to run it without leaking secrets.
2. Add tests around the currently-working happy path so future iteration stays fast instead of fragile.
3. Improve mobile layout and reconnect behavior for intermittent mobile sessions.
4. Make background jobs more visible, inspectable, and resumable.
5. Add lightweight project automation: CI, dependency updates, and SAST.
6. Add only the minimum security/access-control needed for a single-user tailnet-only prototype.
