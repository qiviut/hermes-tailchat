# Hermes Tailchat MVP Plan

> For Hermes: use this as the initial execution roadmap, then iterate.

Goal: ship a localhost-first, tailnet-exposed chat app that can talk to Hermes asynchronously.

Architecture:
- FastAPI backend owns persistence and SSE
- Local Hermes API server runs separately and executes the agent
- Static web UI consumes backend APIs only

Phases:
1. MVP: local chat, persistent transcripts, async background run handling, SSE updates
2. Security: tailnet auth, per-user sessions, safer exposure
3. Workflow: approval prompts, persistent jobs, server-initiated messages
4. Operations: systemd units, backups, tests, release process
